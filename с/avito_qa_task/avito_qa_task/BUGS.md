# Найденные баги и особенности API

В процессе тестирования замечены несоответствия общепринятым практикам REST API:

### 1. GET /item/{id} возвращает массив
При запросе конкретного объявления по уникальному ID ожидается возврат одного объекта JSON.
Фактически API возвращает **массив**, содержащий один объект.
*Пример ответа:* `[{ "id": "...", ... }]`

### 2. GET /statistic/{id} возвращает массив
Аналогичная ситуация с методом получения статистики. Ожидается объект статистики, возвращается список из одного элемента.
